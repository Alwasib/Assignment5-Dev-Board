Question-2: What is the difference between innerHTML, innerText, and textContent ?

The Difference between, "innerHTML" retrieves the complete HTML content within an element, including tags, while "innerText" only retrieves the visible text content of an element (ignoring HTML tags), and "textContent" retrieves all text content within an element, including hidden text, but without interpreting any HTML tags; essentially, "innerText" shows what the user sees on the page, while "textContent" gives you the raw text structure regardless of styling or visibility.

Question-3: What is event delegation in the DOM?

In the DOM, "event delegation" is a JavaScript design pattern where you attach a single event listener to a parent element instead of adding listeners to multiple child elements, allowing the parent to handle events that occur on any of its children by leveraging the "event bubbling" mechanism where events propagate from the target element up the DOM tree to its ancestors; this is particularly useful when dealing with a large number of dynamically added child elements, optimizing performance by only needing to manage one event listener.

Question-4: What is event bubbling in the DOM?

Bubbling in the DOM is an event propagation method. It starts with an event triggering on the innermost element and then proceeds to trigger the same event on its parent element and then its parent's parent element and so on.

Question-5: How do you create, add, and remove elements using JavaScript?

1. Create an element: let div = document.createElement("div");
2. Add to the page: document.body.appendChild(div);
3. Remove it: div.remove();
